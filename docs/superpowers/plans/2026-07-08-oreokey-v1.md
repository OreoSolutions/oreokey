# OreoKey v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bộ gõ tiếng Việt macOS (Telex/VNI) dạng menu bar, Rust core + Swift UI, chống dính/nháy chữ bằng chiến lược AX-first.

**Architecture:** Engine gõ thuần túy trong Rust (raw keystroke buffer → re-render → diff tối thiểu → `Action`), tầng platform Rust giữ CGEventTap và bơm sửa chữ (AX API trước, key injection sau), lớp vỏ Swift chỉ làm menu bar + Settings + onboarding, nói chuyện qua C ABI.

**Tech Stack:** Rust (crate `oreokey-core`, staticlib + rlib), Swift 5.9+ (SPM executable + AppKit/SwiftUI), core-foundation/core-graphics crates (MIT/Apache), Sparkle (MIT) — *không dependency GPL*.

## Global Constraints

- macOS 13+; universal binary (aarch64 + x86_64)
- Closed source → mọi dependency phải MIT/Apache-2.0/BSD; cấm tham khảo code GPL (OpenKey, vi-rs, goxkey)
- Luồng nóng xử lý phím nằm trọn trong Rust; Swift không đụng key path
- Engine (`core/src/engine/`) không được import gì ngoài `std`
- Config JSON tại `~/Library/Application Support/OreoKey/`; Rust là chủ duy nhất
- Commit sau mỗi task; test engine chạy bằng `cargo test` không cần quyền hệ thống

### Task 2: Kiến trúc re-render + diff (xương sống engine)

**Files:** Create `core/src/engine/syllable.rs`; Modify `mod.rs`

Cơ chế: giữ `raw: String` (phím ASCII gốc của từ hiện tại). Mỗi phím chữ: `raw.push(c)` → `render(raw)` → so với `last_render` → cắt tiền tố chung → `Replace { backspaces: last_render.chars_kể_từ_điểm_khác, text: phần_mới }`. Backspace: pop raw, re-render tương tự nhưng để app tự xóa 1 ký tự (engine chỉ cập nhật state, trả PassThrough nếu diff đúng bằng 1 ký tự cuối). WordBreak: chốt từ, reset.

`syllable.rs`:
```rust
pub struct Syllable { pub initial: String, pub vowel: String, pub final_c: String, pub tone: Tone, pub horn_breve: …, pub circumflex: … }
pub enum Tone { None, Acute, Grave, Hook, Tilde, Dot }
pub fn parse(raw: &str, method: &dyn RuleSet) -> Rendered; // tách modifier khỏi chữ cái, dựng âm tiết
pub fn place_tone(vowel: &str, tone: Tone, modern: bool) -> String; // luật đặt dấu
```

Luật đặt dấu (bắt buộc đúng): ưu tiên nguyên âm có dấu mũ/móc (ê ơ ô ă â ư); `iê/yê/uô/ươ` → dấu trên nguyên âm thứ hai; kiểu mới `oà/oé/uỷ` đặt trên nguyên âm cuối, kiểu cũ `òa/óe/ủy`; có phụ âm cuối → dấu trên nguyên âm chính sát cuối.

- [ ] Viết diff helper + golden test khung: gõ `v,i,e,t` → mỗi phím PassThrough (chưa có luật)
- [ ] Commit `feat(core): re-render + diff skeleton`

### Task 9: Event tap + key injection (tầng 2+3)

**Files:** Create `core/src/platform/event_tap.rs`, `inject.rs`, `mod.rs`

- `CGEventTap` (crate `core-graphics`, MIT/Apache) tại `kCGHIDEventTap`, listen keyDown + flagsChanged; chạy trên thread riêng có CFRunLoop.
- Map keycode→char theo layout hiện tại (`UCKeyTranslate`); modifier ⌘/⌃ → reset buffer + PassThrough.
- `Replace{n,s}`: nuốt event gốc; post n event Backspace (down+up) rồi **một** event keyDown với `CGEventKeyboardSetUnicodeString(s)` — tất cả post vào `kCGHIDEventTap` với `CGEventTapPostEvent` từ callback để giữ thứ tự; delay vi mô tùy profile.
- Hotkey toggle (so khớp mod+key từ settings) → đổi enabled, gọi status_cb.
- Secure input (`IsSecureEventInputEnabled`) → PassThrough. `tapDisabledByTimeout` → `CGEventTapEnable` lại.
- Backspace do engine bơm phải được tap **bỏ qua** (đánh dấu bằng `CGEventSetIntegerValueField(event, kCGEventSourceUserData, MAGIC)`).

- [ ] Viết + build; test tay bằng binary demo `cargo run --example tapdemo` (in ra key, thử gõ `vieejt`→`việt` trong TextEdit)
- [ ] Commit `feat(platform): event tap + injection`
